# Open First

This repository is the unified workspace for the Sovereign Monad ecosystem.

If you have just opened the root folder, read the docs in this order:

1. `README.md`
2. `CONTRIBUTING.md`
3. `docs/PROJECT_STATE.md`
4. `docs/REPO_STRUCTURE_MAP.md`
5. `docs/SOVEREIGN_MONAD_ECOSYSTEM_MASTER_OPERATING_FILE_v2.4.0.md`

If you are about to change architecture, build order, or sync behavior, also read:

1. `docs/ECOSYSTEM_BUILD_MAP.md`
2. `docs/BUILD_EXECUTION_FLOW.md`
3. `docs/CANONICAL_SYNC_DISCIPLINE.md`

What each tier means:

- `README.md` gives the high-level shape of the repo.
- `docs/PROJECT_STATE.md` is the canonical resume point.
- `docs/REPO_STRUCTURE_MAP.md` tells you what is active, archived, or generated.
- The MOF is the authoritative ecosystem operating state.
- The companion docs explain build sequencing and sync discipline.

Fastest resume path:

```powershell
pnpm status
```

Do not treat archive material as current system truth.
