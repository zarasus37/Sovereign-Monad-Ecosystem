# Build Execution Flow

This document defines the practical order for working in the unified ecosystem.

## Default Flow

1. Read `README.md`.
2. Read `docs/REPO_STRUCTURE_MAP.md`.
3. Read `docs/PROJECT_STATE.md`.
4. Read `docs/SOVEREIGN_MONAD_ECOSYSTEM_MASTER_OPERATING_FILE_v2.4.0.md`.
5. Identify whether the change belongs to an active pillar, a canonical doc, or the archive.
6. Make the smallest change that keeps the unified root coherent.
7. Run `pnpm check:layout` when structural paths or top-level folders change.

## Build Classes

- Zero-capital work: docs, local code organization, verification, and cleanup.
- Capital-gated work: deployment, activation, and live operational claims.

## Rule

Do not let local scaffolds be mistaken for live status.
