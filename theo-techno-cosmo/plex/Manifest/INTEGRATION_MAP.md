## Role-Based Reading Paths

### PATH 1: Builder / Agent Developer

**Goal**: Build agents that operate authentically within the ecosystem

**Reading Sequence**:
1. MOF v2.4.0 Sections 1-2 (philosophy, axioms) — 30 min
2. OPERATIONAL_AXIOMS_PHASE4.md (which axioms are currently active?) — 45 min
3. **AGENT_PERSONALITY_FRAMES_v5.md** (pick your personality archetype) — 1 hour
4. plex/Review/ (case studies of authentic vs. hollow convergence) — 1.5 hours
5. DOVE_OPERATIONAL_SPECIFICATION_v1.md (what Dove watches for) — 45 min

**Why this order**: Understand philosophy first, then your personality role, then what gets measured, then what governance looks for.

- What does "authentic operation" mean for my agent type?

## Data Flow Across Documents

### Input Data Sources

```
Live System State (Monad Ecosystem agents, market data)
           ↓
        LOGOC Scoring (Gnostic Engine)
           ↓
    plex/Review/ Audit Trails
           ↓
  (Three streams converge)
           ↓
    OPERATIONAL_AXIOMS_PHASE4.md
           ↓
 ┌─────────────┬──────────────┬──────────────┐
 ↓             ↓              ↓              ↓
AGENT_         DOVE_          ECOSYSTEM_    GOVERNANCE
PERSONALITY_   OPERATIONAL_   ALIGNMENT_    DECISIONS
FRAMES_v5.md   SPECIFICATION_ AUDIT_PHASE4.
                v1.md         md
```

**What flows where**:
1. **Monad Ecosystem** generates live agent behavior, trades, decisions
2. **Gnostic Engine** processes behavior through LOGOC, generates scores and patterns
3. **plex/Review/** documents patterns as audit trail
4. **OPERATIONAL_AXIOMS_PHASE4.md** interprets whether current behavior aligns with axioms
5. **AGENT_PERSONALITY_FRAMES_v5.md** uses this to guide builder behavior
6. **DOVE_OPERATIONAL_SPECIFICATION_v1.md** uses this to guide governance actions
7. **ECOSYSTEM_ALIGNMENT_AUDIT_PHASE4.md** validates all three pillars stay aligned
8. Feedback loops: governance decisions → new behavior → new scores → new audit trails
